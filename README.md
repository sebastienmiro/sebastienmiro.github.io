README.md
=========

À propos de ce dépôt
--------------------

Ce dépôt contient le code source de mon blog personnel, consacré à la sécurité de l’information par le prisme de l’identité.

Les contenus publiés portent principalement sur :

*   l’identité et les accès,
    
*   la sécurité cloud,
    
*   la gouvernance,
    
*   les environnements Microsoft (Entra ID, Microsoft 365, Conditional Access, Intune, etc.),sans exclusivité technologique.
    
Le blog reflète une pratique terrain, à la frontière entre gouvernance et technique, avec des analyses, des décryptages, des retours d’expérience et des outils concrets.

La ligne éditoriale est volontairement factuelle, sans storytelling inutile ni posture marketing.

Contenu du blog
---------------

Les articles sont organisés selon trois axes complémentaires :

*   **Catégories**Elles décrivent l’intention éditoriale de l’article (analyse, gouvernance, technique, outils).
    
*   **Tags**Ils décrivent les sujets techniques abordés (Entra ID, MFA, tokens, RBAC, passkeys, etc.).
    
*   **Séries**Certaines thématiques sont structurées sous forme de séries (ex. Conditional Access Framework, Un risque / une mesure).
    
Des champs complémentaires (scope, platform) sont utilisés pour améliorer la lisibilité et la navigation, notamment via la sidebar.

Les règles éditoriales sont documentées dans le fichier CONTENT-GUIDELINES.md.

Stack technique
---------------

*   Générateur de site statique : **Jekyll**    
*   Hébergement : **GitHub Pages**    
*   Thème : fork de **Beautiful Jekyll**, adapté et personnalisé    
*   Langage principal : **Markdown**    
*   Contenus versionnés avec le code
    
Aucune dépendance serveur ou base de données.

Objectifs du dépôt
------------------

Ce dépôt a pour objectifs :

*   de versionner le contenu du blog,    
*   de garantir la traçabilité des évolutions,    
*   de documenter les choix éditoriaux et techniques,    
*   de servir de base à des contributions ponctuelles (scripts, exemples, correctifs).    

Ce dépôt n’a pas vocation à être un thème Jekyll générique.

Contributions
-------------

Les contributions ne sont pas ouvertes par défaut.

Des correctifs techniques ou des suggestions peuvent toutefois être proposés via des issues ou des pull requests, à condition qu’ils respectent la ligne éditoriale du blog.
